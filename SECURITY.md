# Segurança

Este é um site estático e público. Nenhum arquivo enviado ao navegador deve
conter senhas, tokens, chaves de API, credenciais ou dados pessoais privados.

## Como reportar

Não publique informações sensíveis em uma issue. Entre em contato com o
responsável pelo repositório por um canal privado e informe:

- o arquivo ou a página afetada;
- uma descrição objetiva do problema;
- passos para reproduzir, sem incluir credenciais reais.

## Se um segredo for exposto

Remover o valor do arquivo não é suficiente, pois ele pode continuar no
histórico do Git. A credencial deve ser revogada e substituída imediatamente;
depois, o histórico do repositório deve ser reescrito, se necessário.
